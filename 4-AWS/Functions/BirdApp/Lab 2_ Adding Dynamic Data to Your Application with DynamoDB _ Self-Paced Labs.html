        Task 7: Using a GSI in a DynamoDB query operation

        Now that you have created a GSI on the BirdSightings table, you can use it in a query operation.

        Earlier in this lab, you used the scan method, which read all of the records in the table before processing them and returning them to the client. However, you don’t always need to read every record in the table. If you know the key or keys that you want to search with, you can use the query method. This method limits the number of reads against your DynamoDB table by only reading the partition for the index that matches the value. By limiting the number of reads against your table, database calls are faster, and you save money.

        1. Review the ddb_query.js script to understand what it does:

        • Return to the browser tab where the AWS Cloud9 IDE is open.

        • In the Environment window, expand the node_server folder.

        • Locate and open the ddb_query.js file.

        • Review the code.

        This code is similar to the script that you used to load records at the beginning of the lab. Notice the same lines of code that import the AWS SDK and configure the document client.

        However, this script has a few differences, such as in the following code snippet.

        var params = {
        TableName: "BirdSightings",
        IndexName: "<index-name>",
        KeyConditionExpression: "#class = :class and #date > :begin_date",
        ExpressionAttributeNames:{
            "#class": "<index-partition-key>",
            "#date": "<index-sort-key>"
        },
        ExpressionAttributeValues: {
            ":class": "3rd Grade",
            ":begin_date": begin_date
